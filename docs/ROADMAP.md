# Roadmap

## Phase 0: Project Framing

- document the vision, v1 scope, research questions, and data principles
- remove obsolete prototype artifacts that no longer reflect the project direction
- avoid premature implementation commitments

## Phase 1: Entity Registry And Taxonomy

- define the segment taxonomy
- version the role, facility, item, and predicate vocabularies
- curate the first 200 companies
- normalize names, aliases, identifiers, and countries
- establish the canonical company universe for v1
- identify the first recurring source spine and the first critical facilities set

## Phase 2: Graph Model And Ingestion

- draft the first real graph model for v1
- define evidence and provenance handling
- separate entities, observations, and claims cleanly enough to support rebuilds
- ingest company, country, role, and selected facility data
- stand up the first recurring adapters for high-operability sources
- add initial dependency and policy-risk relationships

## Phase 3: Baseline Analytics

- concentration analysis by segment and geography
- chokepoint ranking
- dependency path exploration
- disruption and blast-radius analysis

## Phase 4: Research Experience

- improve the usability of the graph for interactive exploration
- create notebooks, dashboards, or query packs that surface insights clearly
- tighten the presentation so the project is demo-ready

## Phase 5: Productization Paths

Possible future directions include:

- researcher-facing SaaS
- managed supply-chain intelligence service
- alerting and monitoring product
- historical change-tracking and scenario analysis workflows

These are intentionally future-facing. The first job is still to build a credible v1 research asset.
